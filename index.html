quickstart
import jax.numpy as jnp
from snnkit.core.neuron import LIFParams, simulate_lif

params = LIFParams(tau=20e-3, v_th=1.0, dt=1e-3)
i_trace = jnp.full((1000,), 1.5)
v, spikes = simulate_lif(i_trace, params)

print(f"{spikes.sum()} spikes / 1000 steps")
# 45 spikes / 1000 steps

DESIGN PHILOSOPHY

Dynamics and structure are kept separate.

Core is plain functions

Neuron dynamics are arrays in, arrays out — JIT-compiles cleanly, no hidden state.

The object API is a thin wrapper

NeuronGroup handles bookkeeping and equation parsing only — every number still comes from core.

One sparse graph format, two use cases

The same SparseWeights built for synaptic current injection holds a real biological connectome, unmodified.

src/snnkit/api/neuron_group.py
from snnkit.api.neuron_group import NeuronGroup

group = NeuronGroup(
    n=100,
    equation="dv/dt = (I - v) / tau",
    v_th=1.0,
    v_reset=0.0,
    dt=1e-3

SCOPE, STATED PLAINLY

What's shipped, what's tracked, what isn't here.

Shipped

  • Core engine: LIF, Izhikevich, sparse synapses, delays, parser
  • BPTT + STDP + SuperSpike training, benchmarked across seeds
  • Real connectome extension, validated diffusion model
  • 50 tests, CI on every push, clean-install verified

Tracked, not dropped

  • E-prop — scoped as a stretch goal, not attempted
  • Real SHD dataset swap-in (synthetic stand-in used here)
  • API stability policy, MkDocs site, experiment logging

Not in this repo

  • Customer discovery, positioning, fundraising — real-world actions, not code
  • Hardware deployment layer — future, funded phase
  • Any clinical or biological validity claim for the connectome demo
